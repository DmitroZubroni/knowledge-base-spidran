## Карта языков по нишам

| Язык | Основные применения | Особенность |
|------|---------------------|-------------|
| **Java** | Бэкенд, Android, Enterprise | JVM, строгая типизация, Spring |
| **Kotlin** | Бэкенд, Android | Современная Java, лаконичнее |
| **Python** | Data Science, ML, скрипты, бэкенд | Читаемость, огромная экосистема |
| **JavaScript** | Фронтенд (везде), бэкенд (Node.js) | Единственный язык браузера |
| **TypeScript** | Фронтенд, бэкенд | JS + строгая типизация |
| **Go** | Инфраструктура, микросервисы, CLI | Простота, скорость, конкурентность |
| **Rust** | Системное ПО, WebAssembly | Безопасность памяти без GC |
| **SQL** | Базы данных | Стандарт запросов, не Тьюринг-полный |
| **C#** | .NET бэкенд, Unity (игры) | Экосистема Microsoft |
| **Swift** | iOS, macOS | Язык Apple |

## Как выбирают язык в компании

1. **Исторически сложилось** — большая часть кода уже написана на Java, менять нецелесообразно
2. **Задача** — ML-модели пишут на Python, высоконагруженные сервисы — Go/Rust
3. **Экосистема** — нужна конкретная библиотека которая есть только в одной экосистеме
4. **Команда** — нанимать легче на распространённый язык

## Нет объективно "лучшего" языка

Это как спрашивать "какой лучший инструмент" не уточняя для чего.

Правильный вопрос: **"Лучший для конкретной задачи в конкретном контексте?"**

- Enterprise бэкенд → Java / Kotlin + Spring
- ML / Data Science → Python
- Веб-интерфейс → TypeScript + React / Vue
- Высоконагруженный сервис → Go
- Мобильное iOS → Swift
- Кроссплатформенный мобайл → Flutter (Dart) / React Native
