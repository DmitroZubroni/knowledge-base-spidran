## Карта языков по применению

| Язык | Где используют | Почему |
|------|---------------|--------|
| **Java / Kotlin** | Бэкенд, Android | Надёжность, Spring, JVM |
| **Python** | Data Science, ML, скрипты, бэкенд | Простота, библиотеки (NumPy, TensorFlow) |
| **JavaScript** | Фронтенд (везде), бэкенд (Node.js) | Единственный язык браузера |
| **TypeScript** | Фронтенд, бэкенд | JS + строгая типизация |
| **Go** | Инфраструктура, микросервисы | Простота, скорость, конкурентность |
| **Rust** | Системное ПО, WebAssembly | Безопасность памяти + скорость C++ |
| **SQL** | Базы данных | Стандарт запросов к реляционным БД |

---

## JavaScript — особый случай

JS изначально создавался для браузера. Потом появился **Node.js** — среда запуска JS вне браузера — и JS пошёл на сервер.

Сейчас JS/TS — самый популярный язык по числу разработчиков. React, Vue, Angular — всё на JS/TS.

---

## Python — язык для всего

Python читается почти как английский текст. Это делает его идеальным для:
- Быстрых скриптов и автоматизации
- Data Science и ML (NumPy, Pandas, TensorFlow, PyTorch)
- Бэкенда (FastAPI, Django, Flask)

Один из лучших языков для начала — и для вайбкодинга с AI.

---

## Нет "лучшего" языка

Правильный вопрос: **"Лучшего для чего?"**

- Делаешь бэкенд для корпоративного приложения → Java/Kotlin + Spring
- Делаешь ML модель → Python
- Делаешь сайт → TypeScript + React/Vue
- Делаешь CLI утилиту → Go или Python
- Хочешь максимальную скорость → Rust или C++

---

## 🎯 Главное

> Каждый язык занял свою нишу. Нет смысла искать один лучший — нужно понимать для чего какой инструмент. Для бэкенда на JVM — Java/Kotlin. Для ML и скриптов — Python. Для браузера — JS/TS.
