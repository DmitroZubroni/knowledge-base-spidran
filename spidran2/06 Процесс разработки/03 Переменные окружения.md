## Проблема: настройки зависят от окружения

Одно и то же приложение запускается в разных местах:
- На ноутбуке разработчика
- На сервере staging
- На production

У каждого окружения — свои настройки: адрес базы данных, секретные ключи, API-токены. Хранить их прямо в коде — плохая идея.

---

## Хардкод — антипаттерн

```java
// ❌ Плохо — пароль в коде, который попадёт в Git
String dbPassword = "super_secret_123";
String apiKey = "sk-1234567890abcdef";
```

Если этот код попадёт в публичный репозиторий — злоумышленники получат доступ к БД.

---

## Переменные окружения — правильный подход

```java
// ✅ Хорошо — значение берётся из окружения
String dbPassword = System.getenv("DB_PASSWORD");
String apiKey = System.getenv("OPENAI_API_KEY");
```

Переменные окружения задаются **вне кода** — на сервере, в CI/CD, или в `.env` файле локально.

---

## .env файл

При разработке локально удобно использовать файл `.env`:

```bash
# .env — не коммитить в Git!
DB_HOST=localhost
DB_PORT=5432
DB_PASSWORD=local_password
API_KEY=test_key_123
```

**.env файл обязательно добавляется в `.gitignore`** — он не должен попасть в репозиторий.

---

## Типичные переменные окружения

| Переменная | Что хранит |
|-----------|-----------|
| `DATABASE_URL` | Строка подключения к БД |
| `JWT_SECRET` | Секрет для подписи токенов |
| `SMTP_PASSWORD` | Пароль email-сервиса |
| `PORT` | Порт на котором запускается сервер |
| `NODE_ENV` | Окружение: development / production |
| `LOG_LEVEL` | Уровень логирования: debug / info / error |
