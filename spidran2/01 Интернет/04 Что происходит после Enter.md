## Полный путь за ~200 миллисекунд

Ты вводишь `github.com` и нажимаешь Enter. Вот что происходит:

```
1.  DNS lookup          ~20мс   → узнаём IP: 140.82.112.4
2.  TCP handshake       ~30мс   → устанавливаем соединение
3.  TLS handshake       ~60мс   → договариваемся о шифровании
4.  HTTP GET /          ~10мс   → запрашиваем главную страницу
5.  Сервер отвечает     ~20мс   → отдаёт HTML (200 OK)
6.  Браузер парсит HTML         → видит ссылки на CSS, JS, картинки
7.  Параллельные GET            → загружает все ресурсы
8.  Рендеринг                   → строит страницу на экране
```

## Браузер — это сложная платформа

Современный браузер включает:

| Компонент | Функция |
|-----------|---------|
| **Сетевой стек** | Управляет HTTP-запросами |
| **HTML-парсер** | Читает разметку, строит DOM-дерево |
| **CSS-движок** | Применяет стили к элементам |
| **JS-движок** (V8, SpiderMonkey) | Выполняет JavaScript |
| **Движок рендеринга** | Отрисовывает итоговую картинку |
| **Кеш** | Хранит ресурсы, чтобы не загружать повторно |

## DevTools — смотрим под капот

Нажми `F12` → вкладка **Network**. Здесь виден каждый запрос:
- URL и метод
- Статус-код ответа
- Размер и время загрузки
- Заголовки запроса и ответа

Это главный инструмент для понимания что происходит между браузером и сервером.

---

## Кеш браузера

Браузер сохраняет загруженные ресурсы (CSS, JS, картинки) локально. При следующем визите — берёт из кеша, не загружает заново.

Сервер управляет кешированием через заголовки:
```
Cache-Control: max-age=86400    ← кешировать на 24 часа
Cache-Control: no-cache         ← всегда проверять актуальность
```
