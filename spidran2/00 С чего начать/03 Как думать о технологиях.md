## Стек технологий

**Технологический стек** — набор технологий которые используются для создания одного продукта. У каждого слоя своя роль.

Типичный стек веб-приложения:

```
┌─────────────────────────────────┐
│     Браузер (Chrome, Safari)    │  ← пользователь видит это
├─────────────────────────────────┤
│   Фронтенд (React / Vue / HTML) │  ← интерфейс
├─────────────────────────────────┤
│    Бэкенд (Java Spring / Node)  │  ← логика и API
├─────────────────────────────────┤
│  База данных (PostgreSQL/Redis)  │  ← хранение данных
├─────────────────────────────────┤
│  Инфраструктура (Docker / AWS)  │  ← где всё это живёт
└─────────────────────────────────┘
```

---

## Почему технологий так много

Потому что каждая решает конкретную задачу. Нет "лучшей технологии" — есть "подходящая для задачи".

**Аналогия:** в кухне есть нож, тёрка, блендер и духовка. Не потому что производители не смогли сделать одно устройство — а потому что каждый инструмент оптимален для своего.

---

## Абстракции — главный принцип IT

Каждый слой стека **скрывает сложность нижнего слоя**. Это называется абстракция.

- Ты пишешь `println("Привет")` — не думаешь о битах
- Spring обрабатывает HTTP — ты не думаешь о TCP-пакетах
- PostgreSQL хранит данные — ты не думаешь о дисковых секторах

Понять IT = понять что скрывает каждый слой и зачем он нужен.

---

## Главное правило курса

> Технологии меняются. Принципы — нет.

HTTP существует с 1991 года. Git — с 2005. Реляционные базы данных — с 1970-х. Ты изучаешь принципы, которые будут актуальны через 20 лет, а не конкретные инструменты которые могут устареть.
