## HTML — это разметка, не программирование

**HTML (HyperText Markup Language)** — язык разметки, описывает структуру страницы. Это не код с логикой — это описание "что здесь есть".

**Аналогия:** HTML — это каркас здания. Вот где стены, вот где окна, вот где дверь. Внешний вид и цвет — это уже CSS.

---

## Теги — строительные блоки

HTML состоит из тегов — именованных блоков:

```html
<h1>Заголовок страницы</h1>
<p>Абзац с текстом.</p>
<img src="photo.jpg" alt="Фото">
<a href="https://google.com">Ссылка</a>
<button>Нажми меня</button>
```

Большинство тегов парные: открывающий `<p>` и закрывающий `</p>`.

---

## DOM — дерево страницы

Браузер читает HTML и строит **DOM (Document Object Model)** — дерево объектов:

```
document
  └── html
        ├── head
        │     └── title: "Моя страница"
        └── body
              ├── h1: "Привет"
              └── p: "Текст"
```

JavaScript работает именно с этим деревом — добавляет, удаляет, изменяет узлы.

---

## Семантика важна

Теги несут смысл, а не только вид:

| Тег | Смысл |
|-----|-------|
| `<header>` | Шапка страницы |
| `<nav>` | Навигация |
| `<main>` | Основной контент |
| `<article>` | Самостоятельная статья |
| `<footer>` | Подвал страницы |
| `<button>` | Кнопка (интерактивная) |

Правильная семантика помогает поисковикам, скринридерам и разработчикам понимать страницу.
