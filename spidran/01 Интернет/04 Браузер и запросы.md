## Что происходит после Enter

Ты вводишь `github.com` и нажимаешь Enter. Вот что происходит за ~200мс:

```
1. DNS lookup       → находим IP адрес github.com
2. TCP handshake    → устанавливаем соединение с сервером
3. TLS handshake    → договариваемся о шифровании (HTTPS)
4. HTTP GET /       → просим главную страницу
5. Сервер отвечает  → отдаёт HTML
6. Браузер парсит   → находит ссылки на CSS, JS, картинки
7. Доп. запросы     → загружает CSS, JS, шрифты
8. Рендеринг        → строит страницу на экране
```

---

## Браузер — это не просто "смотрелка"

Современный браузер — это мощная платформа:
- **HTML-парсер** — читает разметку, строит DOM-дерево
- **CSS-движок** — применяет стили
- **JavaScript-движок** (V8 в Chrome) — выполняет код
- **Сетевой стек** — управляет HTTP-запросами
- **Кеш** — хранит ресурсы чтобы не загружать повторно

---

## DevTools — твой лучший друг

Нажми `F12` в браузере. Вкладка **Network** — здесь видно каждый запрос:
- Куда он идёт
- Какой метод (GET/POST)
- Статус ответа
- Сколько времени занял
- Что именно вернул сервер

Это один из главных инструментов для понимания что происходит в приложении.

---

## Куки и хранилище

Браузер умеет хранить данные на устройстве:
- **Cookie** — маленькие данные, отправляются с каждым запросом (используются для сессий)
- **localStorage** — данные остаются после закрытия браузера
- **sessionStorage** — очищается при закрытии вкладки

Именно в куки обычно хранится твой токен авторизации — чтобы не логиниться заново.
