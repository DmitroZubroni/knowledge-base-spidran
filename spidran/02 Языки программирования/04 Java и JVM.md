## Почему Java до сих пор №1 в enterprise

Java существует с 1995 года и не собирается уходить. Причины:

- **Стабильность** — обратная совместимость. Код написанный 20 лет назад работает сегодня
- **JVM** — огромная экосистема всего что работает на виртуальной машине (Kotlin, Scala, Groovy)
- **Spring Framework** — стандарт для бэкенда в корпоративной разработке
- **Миллиарды устройств** — Android долгое время работал на Java

---

## JVM — виртуальная машина Java

**JVM (Java Virtual Machine)** — это программа, которая запускает Java-программы. Она:
- Выполняет байткод
- Управляет памятью (Garbage Collector убирает ненужные объекты)
- Оптимизирует код во время работы (JIT-компиляция)

```
.java файл → javac → .class (байткод) → JVM → работает
```

**Garbage Collector (GC)** — автоматическая уборка памяти. В C тебе надо самому освобождать память. В Java JVM делает это за тебя.

---

## JDK, JRE, JVM — что что

- **JVM** — виртуальная машина, запускает программы
- **JRE** (Java Runtime Environment) — JVM + стандартные библиотеки (для запуска)
- **JDK** (Java Development Kit) — JRE + компилятор + инструменты (для разработки)

Если ты разрабатываешь — нужен JDK. Если только запускаешь — JRE.

---

## Kotlin — современная Java

**Kotlin** — язык от JetBrains, работает на той же JVM. Более лаконичный, безопасный, с меньшим количеством шаблонного кода. Полностью совместим с Java-кодом.

Google официально рекомендует Kotlin для Android. В бэкенд разработке всё больше проектов переходят с Java на Kotlin.

---

## 🎯 Главное

> JVM — платформа на которой работает Java и Kotlin. GC управляет памятью автоматически. Spring + Java/Kotlin = стандарт enterprise бэкенда.
