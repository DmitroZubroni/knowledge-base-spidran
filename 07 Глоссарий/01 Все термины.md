# 📖 Все термины

> [!abstract] Навигация

---

## A

**API (Application Programming Interface)** — интерфейс через который программы общаются друг с другом. [[05 API и REST]]

**Authentication (Аутентификация)** — проверка личности пользователя ("кто ты?"). [[05 Безопасность]]

**Authorization (Авторизация)** — проверка прав ("что тебе можно?"). [[05 Безопасность]]

---

## B

**Backend** — серверная часть приложения: логика, база данных, API. [[01 Клиент и сервер]]

**Bytecode (Байткод)** — промежуточный формат кода Java/Kotlin, выполняемый JVM. [[02 Компиляция и интерпретация]]

**Branch (Ветка)** — параллельная линия разработки в Git. [[01 Git и версионирование]]

---

## C

**CI/CD (Continuous Integration / Delivery)** — автоматическая сборка, тестирование и деплой при каждом коммите. [[03 CI CD и деплой]]

**Client (Клиент)** — часть приложения на устройстве пользователя: браузер, мобильное приложение. [[01 Клиент и сервер]]

**Commit (Коммит)** — снимок состояния кода в Git с описанием изменений. [[01 Git и версионирование]]

**Container (Контейнер)** — изолированная среда для запуска приложения со всеми зависимостями. [[02 Контейнеры и Docker]]

**CORS** — механизм браузера, ограничивающий запросы с других доменов. [[05 Безопасность]]

---

## D

**Deploy (Деплой)** — процесс публикации новой версии приложения на сервере. [[03 CI CD и деплой]]

**DNS (Domain Name System)** — система перевода доменных имён в IP-адреса. [[03 DNS и домены]]

**DOM (Document Object Model)** — дерево объектов HTML-страницы, с которым работает JS. [[01 HTML — структура]]

---

## E

**Environment (Окружение)** — среда запуска приложения: local, dev, staging, production. [[01 Серверы и облако]]

**Environment Variables (Переменные окружения)** — конфигурация приложения вне кода. [[03 Переменные окружения]]

---

## F

**Framework (Фреймворк)** — готовый каркас для приложения, в который вставляешь свой код. [[03 Фреймворки и зависимости]]

**Frontend** — клиентская часть приложения: HTML, CSS, JavaScript. [[main Фронтенд]]

---

## G

**GC (Garbage Collector)** — автоматическое освобождение неиспользуемой памяти. [[02 Компиляция и интерпретация]]

**Git** — система контроля версий кода. [[01 Git и версионирование]]

**GraphQL** — альтернатива REST, где клиент сам указывает нужные поля. [[03 API как контракт]]

---

## H

**HTTP** — протокол передачи данных между браузером и сервером. [[02 Протоколы и HTTP]]

**HTTPS** — HTTP с шифрованием TLS. [[02 Протоколы и HTTP]]

---

## I

**IDE** — среда разработки с автодополнением, отладчиком, интеграцией с Git. [[02 IDE и инструменты]]

**Index (Индекс)** — структура данных для ускорения поиска в БД. [[02 Базы данных]]

**IP-адрес** — уникальный числовой адрес устройства в сети. [[01 Сети и пакеты]]

---

## J

**JSON** — формат данных для обмена между клиентом и сервером. [[05 API и REST]]

**JWT (JSON Web Token)** — токен для авторизации пользователя. [[05 Безопасность]]

**JVM (Java Virtual Machine)** — виртуальная машина, выполняющая байткод Java/Kotlin. [[02 Компиляция и интерпретация]]

---

## K

**Kubernetes (K8s)** — система оркестрации контейнеров. [[02 Контейнеры и Docker]]

---

## L

**Library (Библиотека)** — готовый набор функций, которые вызываешь в своём коде. [[03 Фреймворки и зависимости]]

**Logs (Логи)** — записи о событиях в работающем приложении. [[04 Как читать ошибки]]

---

## M

**Microservices (Микросервисы)** — архитектура, где каждая функция — отдельный сервис. [[04 Монолит и микросервисы]]

**Monolith (Монолит)** — приложение как единый развёртываемый блок. [[04 Монолит и микросервисы]]

---

## N

**NoSQL** — базы данных без жёсткой схемы: MongoDB, Redis, Cassandra. [[02 Базы данных]]

---

## O

**ORM (Object-Relational Mapping)** — инструмент для работы с БД через объекты кода. (пример: Hibernate)

---

## P

**Package Manager** — менеджер зависимостей: npm, Maven, pip. [[03 Фреймворки и зависимости]]

**Protocol (Протокол)** — набор правил обмена данными между участниками. [[02 Протоколы и HTTP]]

**Pull Request (PR)** — запрос на проверку и слияние ветки в Git. [[01 Git и версионирование]]

---

## R

**Repository (Репозиторий)** — проект с полной историей изменений в Git. [[01 Git и версионирование]]

**REST** — архитектурный стиль для построения API поверх HTTP. [[05 API и REST]]

---

## S

**Server (Сервер)** — компьютер в датацентре, обрабатывающий запросы. [[01 Серверы и облако]]

**SQL** — язык запросов к реляционным базам данных. [[02 Базы данных]]

**SSL/TLS** — протоколы шифрования для HTTPS. [[05 Безопасность]]

**Stack trace** — цепочка вызовов кода, приведшая к ошибке. [[04 Как читать ошибки]]

---

## T

**TCP/IP** — базовые протоколы интернета: надёжная доставка пакетов. [[01 Сети и пакеты]]

**TDD** — разработка через тесты: сначала пишешь тест, потом код. [[05 Тестирование]]

---

## V

**VM (Virtual Machine)** — программная эмуляция компьютера. [[01 Серверы и облако]]

---

## W

**WebSocket** — протокол для постоянного двустороннего соединения (чаты, real-time).
