## JS делает страницу живой

HTML — структура. CSS — внешний вид. **JavaScript — поведение и интерактивность**.

Без JS страница — статичный документ. С JS — приложение:
- Нажал кнопку → открылось меню
- Ввёл в поиск → список фильтруется в реальном времени
- Прокрутил вниз → подгрузились новые посты

## JS — единственный язык браузера

Это исторически сложившийся факт: браузеры умеют выполнять только JavaScript. TypeScript, CoffeeScript и другие языки **компилируются в JS** перед запуском.

Исключение — WebAssembly, но это отдельная история.

## Что умеет JS в браузере

```javascript
// Изменить DOM
document.getElementById('title').textContent = 'Новый заголовок'

// Обработать событие
button.addEventListener('click', () => {
  alert('Нажали!')
})

// Запросить данные с сервера
const response = await fetch('/api/users')
const users = await response.json()
```

---

## Асинхронность — ключевая концепция

Браузер не может "зависнуть" на одном запросе. Пока данные загружаются — пользователь должен мочь кликать кнопки.

**async/await** — современный способ работать с асинхронным кодом:

```javascript
async function loadUser(id) {
  const response = await fetch(`/api/users/${id}`)  // ждём ответа
  const user = await response.json()                 // ждём парсинга
  return user
}
```

`await` говорит: "подожди здесь, но не блокируй всё остальное".

## Node.js — JS вне браузера

В 2009 году появился **Node.js** — среда для запуска JS на сервере. Теперь JS работает и на фронтенде, и на бэкенде.

Это открыло путь к **fullstack-разработке** на одном языке.
