## Разделение обязанностей

Почти каждое современное приложение разделено на две части:

**Клиент (Frontend)** — то, что видит пользователь:
- Браузер (React, Vue, Angular, обычный HTML)
- Мобильное приложение (Android, iOS)
- Десктопное приложение

**Сервер (Backend)** — логика и данные:
- Обрабатывает запросы
- Работает с базой данных
- Хранит бизнес-логику
- Отдаёт данные клиенту через API

---

## Почему не всё на клиенте?

Хороший вопрос. Вот причины держать логику на сервере:

1. **Безопасность** — секретные ключи, пароли, чувствительные данные нельзя слать в браузер
2. **База данных** — к ней нельзя давать прямой доступ извне
3. **Производительность** — сервер мощнее телефона
4. **Один источник правды** — все клиенты (браузер, iOS, Android) получают одни и те же данные

## Схема типичного веб-приложения

```
Браузер (React)
     ↕ HTTP/JSON
  Бэкенд (Spring Boot / Node.js)
     ↕ SQL / ORM
  База данных (PostgreSQL)
```

Бэкенд — это "мозг". Он знает правила: кто может видеть что, как считать цену, когда отправить email.

---

## SSR vs SPA

**SPA (Single Page Application):**  
Браузер загружает один HTML, дальше JS перерисовывает страницу без перезагрузки. React, Vue, Angular — SPA.

**SSR (Server Side Rendering):**  
Сервер формирует HTML и отдаёт готовую страницу. Быстрее первая загрузка, лучше для SEO. Next.js, Nuxt.js.

---

## 🎯 Главное

> Клиент (фронтенд) показывает UI. Сервер (бэкенд) хранит логику и данные. Общаются через API. Логика на сервере — ради безопасности и единого источника правды.
