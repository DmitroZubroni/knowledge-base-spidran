## Почему не просто HTML + JS?

На маленьком сайте — можно. На большом приложении (сотни экранов, сложное состояние) — ручное управление DOM превращается в кошмар.

**Проблема:** при каждом изменении данных нужно вручную обновить нужные элементы страницы. Легко пропустить, легко сломать.

---

## Фреймворк — решение проблемы

Фреймворки предлагают подход: **ты описываешь как страница должна выглядеть при данных X, фреймворк сам обновляет DOM**.

```jsx
// React: описываешь что показать
function UserCard({ user }) {
  return (
    <div>
      <h2>{user.name}</h2>
      <p>{user.email}</p>
    </div>
  )
}
// При изменении user → DOM обновляется автоматически
```

## Три главных фреймворка

| Фреймворк | Автор | Особенность |
|-----------|-------|-------------|
| **React** | Meta (Facebook) | Самый популярный, библиотека (не фреймворк строго) |
| **Vue** | Evan You | Проще в освоении, хорошая документация |
| **Angular** | Google | Полный фреймворк, строгая структура, TypeScript |

Рынок труда: React доминирует в большинстве стран.

## SPA vs SSR vs SSG

**SPA (Single Page Application):**  
Браузер загружает один HTML, JS перерисовывает содержимое без перезагрузки. Быстрая навигация внутри приложения.

**SSR (Server Side Rendering):**  
Сервер формирует HTML для каждого запроса. Быстрее первая загрузка, лучше SEO. Next.js (React), Nuxt (Vue).

**SSG (Static Site Generation):**  
HTML генерируется один раз при сборке. Максимальная скорость, подходит для блогов и документации.
