## API — интерфейс для программ

**API (Application Programming Interface)** — способ которым одна программа обращается к другой.

**Аналогия:** официант в ресторане. Ты не идёшь на кухню сам. Ты делаешь заказ официанту, он передаёт на кухню, приносит результат. Официант — это API между тобой и кухней.

---

## REST — стандарт де-факто для веб-API

**REST** — набор соглашений о том как строить API поверх HTTP. Не технология, не стандарт — просто архитектурный стиль, который все приняли.

Пример: API для работы с пользователями

```
GET    /api/users        → список всех пользователей
GET    /api/users/42     → пользователь с id=42
POST   /api/users        → создать пользователя
PUT    /api/users/42     → обновить пользователя 42
DELETE /api/users/42     → удалить пользователя 42
```

Ресурсы описываются существительными (`users`, `orders`). Действие — HTTP-метод.

## JSON — формат данных

Данные передаются в формате **JSON (JavaScript Object Notation)**:

```json
{
  "id": 42,
  "name": "Дмитрий",
  "email": "d@example.com",
  "active": true,
  "roles": ["user", "admin"]
}
```

Читаемый людьми, понимаемый машинами. Используется в 95% современных API.

## Как это работает в реальном приложении

Когда приложение ВКонтакте загружает ленту:

```
Приложение                      Сервер ВК
    │                                │
    │── GET /api/feed?userId=123 ──→ │
    │                                │ проверяет токен
    │                                │ достаёт посты из БД
    │ ←── 200 OK { posts: [...] } ──│
    │                                │
    отображает посты
```

Фронтенд и бэкенд общаются исключительно через API. Это даёт независимость: можно заменить iOS-приложение на Android, не меняя сервер.
