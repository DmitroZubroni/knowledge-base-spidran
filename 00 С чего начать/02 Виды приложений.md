## Три основных вида

### Веб-приложения
Работают в браузере. Код живёт на сервере, пользователь видит результат через Chrome/Firefox/Safari.

**Примеры:** Gmail, ВКонтакте, GitHub, YouTube  
**Плюс:** работает на любом устройстве с браузером  
**Минус:** нужен интернет, ограниченный доступ к железу

### Мобильные приложения
Установлены на смартфон, работают нативно на iOS или Android.

**Примеры:** Telegram, Instagram, банковские приложения  
**Плюс:** доступ к камере, геолокации, уведомлениям  
**Минус:** нужно публиковать в App Store / Google Play, разные версии для iOS и Android

### Десктопные приложения
Установлены на компьютер, работают без браузера.

**Примеры:** IntelliJ IDEA, Figma, Slack, VS Code  
**Плюс:** полный доступ к ресурсам компьютера  
**Минус:** нужно устанавливать, обновлять, разные версии для Windows/Mac/Linux

## Граница размылась

Сегодня различие не такое чёткое:
- **PWA (Progressive Web App)** — веб-приложение которое можно "установить" на телефон и оно работает офлайн
- **Electron** — десктопное приложение написанное на веб-технологиях (Slack, VS Code — именно так)
- **React Native / Flutter** — пишешь один раз, работает на iOS и Android

## Клиентская и серверная части

Почти любое приложение разделено на две части:

```
[Клиент]                    [Сервер]
Браузер / телефон  ←────→  Компьютер в датацентре
Показывает UI               Хранит данные, выполняет логику
Код на устройстве           Код у разработчика
```

Когда ты открываешь ВКонтакте — твой браузер (клиент) загружает интерфейс и запрашивает данные у серверов ВКонтакте. Сервер отвечает твоими постами, друзьями, сообщениями.
