## Что такое API

**API** (Application Programming Interface) — это набор правил, по которым одна программа может попросить что-то у другой.

Аналогия: ты в ресторане. Ты не идёшь сам на кухню — ты говоришь официанту (API): "Принеси мне пасту". Официант передаёт запрос на кухню и возвращает блюдо. Ты не знаешь, как кухня работает — тебе нужен только результат.

---

## Зачем нужен API

Представь приложение для заказа такси. Ему нужны:
- 🗺️ Карты — оно использует API Google Maps
- 💳 Оплата — API платёжной системы
- 📱 Уведомления — API Firebase
- 🌤️ Погода — API weather-сервиса

Каждый из этих сервисов предоставляет API — способ попросить у него данные или действие.

## REST API — самый популярный формат

**REST** — это стиль проектирования API. Почти все современные веб-сервисы используют его.

Запрос выглядит как адрес:

```
GET    https://api.example.com/users/42        — получить пользователя #42
POST   https://api.example.com/users           — создать нового пользователя
PUT    https://api.example.com/users/42        — обновить пользователя #42
DELETE https://api.example.com/users/42        — удалить пользователя #42
```

Данные обычно передаются в формате **JSON** — это просто текст, структурированный как словарь:

```json
{
  "id": 42,
  "name": "Дмитрий",
  "email": "d@example.com"
}
```

## Итог

- API = способ для программ общаться друг с другом
- REST API = самый распространённый стиль, работает через HTTP
- JSON = формат данных, который при этом передаётся
