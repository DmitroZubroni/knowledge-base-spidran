## Ошибки — это нормально

Код никогда не работает с первого раза. Умение читать ошибки и находить их причину — один из ключевых навыков разработчика.

---

## Stack trace — маршрут ошибки

Когда Java-программа падает, она выводит **stack trace** — цепочку вызовов которая привела к ошибке:

```
Exception in thread "main" java.lang.NullPointerException
    at com.example.UserService.getEmail(UserService.java:42)
    at com.example.OrderService.createOrder(OrderService.java:18)
    at com.example.Main.main(Main.java:7)
```

**Читать снизу вверх:**
1. `Main.main` — отсюда всё началось
2. `OrderService.createOrder` — вызвал UserService
3. `UserService.getEmail:42` — здесь упало

Строка 42 в UserService.java — именно там `NullPointerException`.

## Типичные ошибки

| Ошибка | Значит |
|--------|--------|
| `NullPointerException` | Обратились к объекту который равен null |
| `ClassCastException` | Попытка привести объект к несовместимому типу |
| `ArrayIndexOutOfBoundsException` | Индекс за пределами массива |
| `StackOverflowError` | Бесконечная рекурсия |
| `OutOfMemoryError` | Закончилась память JVM |
| `Connection refused` | Сервис недоступен или не запущен |
| `404 Not Found` | Такого URL не существует |
| `500 Internal Server Error` | Ошибка на сервере (смотри логи сервера) |

## Логи — хроника событий

**Логи** — записи о том что происходит в приложении. Уровни:

```
DEBUG   — детали для разработки (отключают в production)
INFO    — обычные события ("Пользователь авторизован")
WARN    — предупреждения (не критично, но стоит знать)
ERROR   — ошибки требующие внимания
```

```bash
# Смотреть логи в терминале
tail -f application.log

# Найти все ERROR в логах
grep "ERROR" application.log
```

## Алгоритм отладки

```
1. Прочитай ошибку внимательно — часто ответ там
2. Найди строку в stack trace — с неё начинай
3. Посмотри значения переменных в этом месте
4. Воспроизведи ошибку изолированно
5. Погугли текст ошибки — кто-то уже сталкивался
6. Исправь, проверь что тест проходит
```
