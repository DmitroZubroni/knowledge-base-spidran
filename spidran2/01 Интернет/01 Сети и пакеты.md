## Интернет — это буквально провода

Интернет — глобальная сеть физических кабелей. Под океанами лежат оптоволоконные магистрали толщиной с руку. Wi-Fi — это лишь последние несколько метров пути; дальше всё равно кабель.

```
[Твой ноутбук]
      ↓ Wi-Fi (воздух, ~10м)
[Домашний роутер]
      ↓ Кабель провайдера
[Узел провайдера]
      ↓ Оптоволокно
[Магистральный маршрутизатор]
      ↓ Трансокеанский кабель
[Датацентр в другой стране]
```

## Пакеты — данные нарезаются на части

Данные в интернете не передаются цельным куском. Они разбиваются на **пакеты** — фрагменты ~1500 байт каждый.

**Почему так:**
- Если пакет потерялся — пересылается только он, не весь файл
- Разные пакеты могут идти разными маршрутами
- Сеть эффективнее использует каналы

Получатель собирает пакеты обратно в нужном порядке.

## IP-адрес — уникальный адрес устройства

Каждое устройство в интернете имеет **IP-адрес**.

- **IPv4:** `142.250.74.46` — 4 числа от 0 до 255, ~4 млрд адресов (уже заканчиваются)
- **IPv6:** `2001:0db8:85a3::8a2e:0370:7334` — 128 бит, практически бесконечно

Роутеры используют IP чтобы решать куда направить пакет дальше — как почтовые сортировочные центры.

---

## TCP/IP — надёжная доставка

**TCP (Transmission Control Protocol)** гарантирует что:
- Все пакеты дошли
- Дошли в правильном порядке
- Ни один не потерялся

Перед отправкой данных TCP устанавливает соединение ("рукопожатие"):
```
Клиент → SYN → Сервер
Клиент ← SYN-ACK ← Сервер
Клиент → ACK → Сервер
[Соединение установлено]
```
