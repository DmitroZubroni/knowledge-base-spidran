## Сервер — это просто компьютер

Сервер — обычный компьютер без монитора и клавиатуры, работающий круглосуточно в датацентре. Датацентр — здание с тысячами серверов, бесперебойным питанием, охлаждением и интернет-каналами.

---

## Физический сервер vs Виртуальная машина

**Физический сервер (bare metal)** — реальное железо. Максимальная производительность, но долго получить и негибко.

**Виртуальная машина (VM)** — программная эмуляция сервера поверх физического. Один физический сервер делится на десятки VM.

```
Физический сервер
    ├── VM 1 (4 CPU, 8GB RAM)  ← твой сервер
    ├── VM 2 (2 CPU, 4GB RAM)  ← чужой сервер
    └── VM 3 (8 CPU, 16GB RAM) ← ещё чей-то
```

---

## Облако — аренда серверов по требованию

Облачные провайдеры (AWS, Google Cloud, Yandex Cloud) предоставляют серверы "по кнопке":
- Нажал → через минуту работающий сервер
- Нагрузка выросла → добавил серверов
- Нагрузка упала → убрал, платишь только за использование

**Крупные провайдеры:**

| Провайдер | Особенность |
|-----------|-------------|
| **AWS** | Самый большой, максимум сервисов |
| **Google Cloud** | Сильный в ML и аналитике |
| **Azure** | Интеграция с Microsoft |
| **Yandex Cloud** | Россия, соответствие 152-ФЗ |

## Окружения (Environments)

| Окружение | Назначение |
|-----------|-----------|
| **local** | Разработка на ноутбуке |
| **dev** | Тестирование последних изменений |
| **staging** | Финальная проверка перед релизом |
| **production** | Боевой сервер, реальные пользователи |

"Сломалось в проде" = сломалось у реальных пользователей.
