## Программа — это список инструкций

Компьютер ничего не понимает сам по себе. Он просто очень быстро выполняет инструкции, которые ему дают. Программа — это и есть этот список инструкций.

Представь рецепт:
1. Вскипяти воду
2. Добавь макароны
3. Вари 8 минут
4. Слей воду

Это уже похоже на программу: чёткая последовательность шагов, которая приводит к результату.

---

## Почему нельзя писать на русском

Компьютер на самом деле понимает только **нули и единицы** (двоичный код). Программировать напрямую в нулях и единицах — почти невозможно.

Поэтому придумали языки программирования — они позволяют писать инструкции в более читаемом виде, а потом переводят это в то, что понимает процессор.

---

## Алгоритм

**Алгоритм** — это точная последовательность шагов для решения задачи. Код — это реализация алгоритма на конкретном языке.

Например, алгоритм "найти максимум в списке чисел":
1. Возьми первое число как текущий максимум
2. Пройди по всем остальным числам
3. Если текущее число больше максимума — обнови максимум
4. Верни максимум

Это можно написать на любом языке программирования — Python, Java, C++. Алгоритм один, а реализаций — много.

---

## Что делают разработчики

Разработчики берут задачу ("пользователь должен мочь войти в систему"), превращают её в алгоритм, затем записывают этот алгоритм в виде кода на выбранном языке.

Большую часть времени они:
- Читают чужой код
- Отлаживают ошибки (баги)
- Проектируют структуры данных
- Изучают библиотеки и фреймворки

Писать код с нуля — это меньшая часть работы.

---

## Итог

- Код = список инструкций для компьютера
- Языки программирования — посредник между человеком и процессором
- Алгоритм = логика; код = реализация алгоритма
