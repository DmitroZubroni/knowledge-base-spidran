## Что такое протокол

Протокол — это **договорённость о формате общения**. Как правила дорожного движения: все знают, что красный — стоп, зелёный — едем. Если кто-то решит придумать своё — будет хаос.

В интернете таких протоколов много. Каждый отвечает за своё:
- **TCP/IP** — доставка пакетов (базовый уровень)
- **HTTP/HTTPS** — передача веб-страниц
- **SMTP** — email
- **WebSocket** — постоянное соединение в реальном времени

---

## HTTP — язык браузера и сервера

HTTP (HyperText Transfer Protocol) — это протокол по которому браузер **просит** контент, а сервер **отдаёт**.

Запрос выглядит примерно так:
```
GET /index.html HTTP/1.1
Host: example.com
```

Ответ:
```
HTTP/1.1 200 OK
Content-Type: text/html

<html>...</html>
```

---

## Методы HTTP — это глаголы

| Метод | Что делает | Аналогия |
|-------|-----------|---------|
| `GET` | Получить данные | "Дай мне меню" |
| `POST` | Отправить данные | "Я хочу заказать" |
| `PUT` | Обновить целиком | "Замени весь заказ" |
| `PATCH` | Обновить частично | "Добавь соус" |
| `DELETE` | Удалить | "Отмени заказ" |

---

## Статус-коды — это ответы сервера

- `200 OK` — всё хорошо
- `404 Not Found` — страница не существует
- `401 Unauthorized` — не авторизован
- `403 Forbidden` — доступ запрещён
- `500 Internal Server Error` — сервер сломался

---

## HTTPS — HTTP + шифрование

Буква **S** значит Secure. Весь трафик зашифрован с помощью TLS. Без этого твой провайдер, кафе с Wi-Fi или хакер в сети могут читать всё что ты отправляешь.

Замочек в браузере = HTTPS активен.

---

## 🎯 Главное

> HTTP — это язык, на котором браузер просит данные у сервера. HTTPS — то же самое, но зашифрованное. Методы (GET, POST…) говорят серверу что ты хочешь сделать.
