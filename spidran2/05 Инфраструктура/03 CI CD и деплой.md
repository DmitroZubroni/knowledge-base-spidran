## Деплой — доставка кода на сервер

**Деплой** — процесс публикации новой версии приложения на сервере. Раньше это делали вручную: подключались по SSH, останавливали сервер, копировали файлы, запускали. Любая ошибка — downtime.

Сейчас деплой автоматизируют.

---

## CI — Continuous Integration

При каждом коммите в репозиторий **автоматически** запускается:

```
git push
    ↓
[CI сервер]
    ├── Сборка проекта
    ├── Запуск тестов
    ├── Проверка качества кода (линтер)
    └── Результат: ✅ успех / ❌ ошибка
```

Если тесты упали — разработчик сразу видит и чинит. Не копится сломанный код.

---

## CD — Continuous Delivery/Deployment

После успешного CI — автоматический деплой:

```
git push → тесты прошли → задеплоилось на staging
                        → QA проверил
                        → задеплоилось на production
```

**Continuous Delivery** — деплой готов, но требует ручного подтверждения на prod.  
**Continuous Deployment** — деплой на prod происходит полностью автоматически.

## Blue-Green деплой

Техника деплоя без downtime:

```
Сейчас работает:  [Blue  v1.0] ← весь трафик
Готовим:          [Green v1.1]

Переключаем:      [Blue  v1.0]
                  [Green v1.1] ← весь трафик

Если что-то не так — переключаем обратно за секунды.
```
