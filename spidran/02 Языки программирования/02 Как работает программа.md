## Два пути: компиляция и интерпретация

Как код на языке программирования превращается в работающую программу? Есть два основных подхода.

---

## 1. Компиляция — перевести всё заранее

**Компилятор** читает весь твой код и переводит его в машинный код **заранее**, до запуска.

```
Исходный код (.c, .cpp, .rs)
        ↓  компилятор
Машинный код (.exe, бинарник)
        ↓  запуск
Работает очень быстро
```

**Плюс:** программа работает максимально быстро.  
**Минус:** нужно компилировать под каждую платформу отдельно (Windows, Mac, Linux).

Языки: C, C++, Rust, Go.

---

## 2. Интерпретация — переводить на ходу

**Интерпретатор** читает код **во время выполнения**, строка за строкой.

```
Исходный код (.py, .js, .rb)
        ↓  интерпретатор (запущен в момент выполнения)
Выполняется сразу
```

**Плюс:** код работает на любой платформе где есть интерпретатор.  
**Минус:** медленнее, т.к. перевод происходит в реальном времени.

Языки: Python, Ruby, PHP.

---

## 3. Байткод — золотая середина (Java, Kotlin)

Java придумала хитрый путь: **компилируй в промежуточный байткод**, а затем запускай байткод на виртуальной машине (JVM).

```
Java код (.java)
    ↓  javac (компилятор)
Байткод (.class)  ← не машинный код, но не исходник
    ↓  JVM (Java Virtual Machine)
Работает на любой ОС где есть JVM
```

**JVM** — это программа которая умеет выполнять байткод. Она установлена на сервере, на твоём компьютере, везде.

Именно поэтому говорят: **"Write once, run anywhere"** — написал один раз, запускай где угодно.

---

## 🎯 Главное

> Компиляция = перевод заранее (быстро). Интерпретация = перевод на ходу (гибко). Java использует байткод + JVM — компромисс между скоростью и переносимостью.
