## Высокий и низкий уровень

**Уровень** языка = насколько далеко он от железа.

| Уровень | Пример | Ты думаешь о... |
|---------|--------|-----------------|
| Низкий | Assembler, C | ячейках памяти, регистрах процессора |
| Средний | C++, Rust | памяти, но с удобными структурами |
| Высокий | Java, Python, JS | задаче, а не о памяти |

Чем выше уровень — тем быстрее пишешь, тем меньше контроля над железом.

## Статическая vs динамическая типизация

**Тип** — это то, что лежит в переменной: число, строка, список и т.д.

**Статическая типизация** (Java, Kotlin, TypeScript):
```java
String name = "Дмитрий";  // тип указан явно
int age = 25;               // компилятор проверит при сборке
name = 42;                  // ОШИБКА — нельзя положить число в строку
```

**Динамическая типизация** (Python, JavaScript):
```python
name = "Дмитрий"   # тип вычисляется в момент выполнения
name = 42          # окей, переменная теперь число
```

**Статика:** ошибки находятся **до запуска**, код более предсказуем — хорошо для больших проектов.  
**Динамика:** пишется быстрее — хорошо для скриптов и прототипов.

## ООП vs Функциональный подход

**ООП (Объектно-ориентированное программирование):**  
Программа = набор объектов, которые взаимодействуют. У каждого объекта есть данные и поведение.

```java
class User {
    String name;
    void sendEmail() { ... }
}
```

**Функциональный подход:**  
Программа = набор функций. Данные неизменны, функции принимают значения и возвращают новые.

Большинство современных языков поддерживают оба стиля.

> Языки отличаются уровнем абстракции (насколько близко к железу) и типизацией (строгой или нет). Java — высокоуровневый, статически типизированный, ООП-язык.
