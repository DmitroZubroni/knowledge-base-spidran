## Разделение — это осознанное решение

Когда ты открываешь приложение — у кода есть два места где он может жить: на твоём устройстве или на сервере.

**Клиент** — код на устройстве пользователя:
- Браузер (React, Vue, обычный JS)
- Мобильное приложение
- Десктопное приложение

**Сервер** — код в датацентре:
- Обрабатывает запросы от всех клиентов
- Работает с базой данных
- Содержит бизнес-логику
- Работает 24/7

---

## Почему логику держат на сервере

```
❌ Неправильно: проверять скидку на клиенте
   → пользователь может изменить код в браузере
   → поставить любую скидку

✅ Правильно: проверять скидку на сервере
   → пользователь видит только результат
   → изменить нельзя
```

Три причины держать логику на сервере:
1. **Безопасность** — секретные ключи, пароли, правила нельзя отдавать клиенту
2. **Доверие** — сервер — единственный источник правды
3. **Ресурсы** — сервер мощнее телефона

## Типичная схема веб-приложения

```
Пользователь
    ↓
[Браузер / Приложение]    ← Клиент
    ↕ HTTP + JSON
[Бэкенд-сервер]           ← Логика
    ↕ SQL / ORM
[База данных]             ← Данные
```

---

## Масштабирование

Когда пользователей становится много — одного сервера не хватает:

- **Вертикальное масштабирование** — сделать сервер мощнее (больше CPU, RAM)
- **Горизонтальное масштабирование** — добавить больше серверов, распределить нагрузку

Горизонтальное масштабирование сложнее (нужен балансировщик нагрузки, общая БД) но не имеет физического предела.
