## Что такое API

**API (Application Programming Interface)** — это способ, которым одна программа говорит другой: "вот что ты можешь у меня попросить и вот что я тебе отвечу".

**Аналогия:** официант в ресторане — это API. Ты не идёшь сам на кухню (сервер). Ты говоришь официанту (API) что хочешь, он передаёт запрос на кухню и приносит результат.

---

## REST — самый популярный стиль API

**REST** — это набор принципов как строить API поверх HTTP. Не стандарт, не протокол — просто соглашение, которое все приняли.

Пример REST API для работы с пользователями:

| Что делаем | HTTP метод | URL |
|-----------|-----------|-----|
| Получить список | `GET` | `/api/users` |
| Получить одного | `GET` | `/api/users/42` |
| Создать | `POST` | `/api/users` |
| Обновить | `PUT` | `/api/users/42` |
| Удалить | `DELETE` | `/api/users/42` |

---

## JSON — формат данных

Данные в API почти всегда передаются в формате **JSON**:

```json
{
  "id": 42,
  "name": "Дмитрий",
  "email": "d@example.com",
  "roles": ["user", "admin"]
}
```

Читается людьми, понимается машинами. Именно этот формат ты будешь видеть в 95% случаев при работе с любыми сервисами.

---

## Реальный пример

Когда приложение ВКонтакте на телефоне загружает твою ленту:
1. Приложение (клиент) делает `GET https://api.vk.com/feed`
2. Сервер ВК проверяет твой токен, достаёт посты из БД
3. Возвращает JSON с массивом постов
4. Приложение отображает их на экране

Фронтенд и бэкенд общаются именно так — через API.

---

## 🎯 Главное

> API — это интерфейс через который программы общаются. REST — соглашение о том как строить API поверх HTTP. Данные передаются в JSON.
