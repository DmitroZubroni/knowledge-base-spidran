## Деплой — это запуск кода на сервере

**Деплой (deploy)** — процесс доставки кода на сервер чтобы пользователи могли им пользоваться.

Твой ноутбук — не сервер. Код который работает у тебя, должен переехать на машину в датацентре.

---

## Окружения (environments)

Обычно есть несколько окружений:

| Окружение | Для чего | Кто видит |
|-----------|---------|-----------|
| **Local** | Разработка на твоей машине | Только ты |
| **Dev** | Тестирование последних изменений | Команда |
| **Staging** | Финальная проверка перед релизом | Команда + QA |
| **Production (Prod)** | Боевой сервер | Все пользователи |

"Упало в проде" — значит сломалось на боевом сервере у реальных пользователей.

---

## CI/CD — автоматизация деплоя

**CI (Continuous Integration)** — при каждом коммите автоматически:
- Запускаются тесты
- Проверяется качество кода
- Собирается приложение

**CD (Continuous Delivery/Deployment)** — если CI прошёл, автоматически:
- Деплоится на staging
- (или даже на prod, если всё хорошо)

```
git push → GitHub → CI запустил тесты → все прошли
         → CD задеплоил на staging → QA проверил
         → CD задеплоил на production
```

Инструменты: GitHub Actions, GitLab CI, Jenkins.

---

## Облако — арендуй сервер по клику

Вместо покупки физических серверов — арендуй виртуальные в облаке:

- **AWS** (Amazon Web Services) — самый большой
- **Google Cloud (GCP)**
- **Yandex Cloud** — для России

Ты платишь за время работы. Нужно больше мощности — добавляешь инстансов. Нагрузка упала — убираешь. Это **масштабирование**.

---

## 🎯 Главное

> Деплой = код на сервере. Окружения защищают prod от поломок. CI/CD автоматизирует сборку и деплой — без ручных шагов. Облако = аренда серверов по требованию.
