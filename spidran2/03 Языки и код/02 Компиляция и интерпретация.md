## Два способа запустить код

### Компиляция — перевести заранее

**Компилятор** переводит весь исходный код в машинный код **до запуска**. Результат — исполняемый файл.

```
Исходник (.c, .cpp, .rs)
        ↓  компилятор
Бинарный файл (.exe / без расширения)
        ↓  запуск
Работает максимально быстро
```

**Языки:** C, C++, Rust, Go  
**Плюс:** высокая скорость выполнения  
**Минус:** нужно компилировать отдельно для Windows, Mac, Linux

---

### Интерпретация — переводить на ходу

**Интерпретатор** читает и выполняет код строка за строкой во время работы.

```
Исходник (.py, .rb)
        ↓  интерпретатор (работает в момент выполнения)
Выполняется построчно
```

**Языки:** Python, Ruby  
**Плюс:** один файл — запускается везде где есть интерпретатор  
**Минус:** медленнее компилируемых

---

### Байткод — компромисс (Java, Kotlin)

Java компилирует в **байткод** — промежуточный формат — а затем байткод выполняет **JVM (Java Virtual Machine)**.

```
Java (.java)
    ↓  javac
Байткод (.class)   ← не машинный код, платформонезависимый
    ↓  JVM
Выполняется на любой ОС с установленной JVM
```

**"Write once, run anywhere"** — написал один раз, запускай где угодно.

**JIT-компиляция:** современные JVM компилируют "горячие" части байткода в машинный код прямо во время выполнения — получая скорость близкую к C++.

## Сборка мусора (Garbage Collection)

В языках вроде C разработчик вручную выделяет и освобождает память. Забыл освободить — **утечка памяти**.

Java, Python, Go и большинство современных языков имеют **Garbage Collector** — автоматическую уборку неиспользуемых объектов из памяти.
