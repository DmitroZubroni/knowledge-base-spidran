## Проблема: люди плохо запоминают числа

IP-адрес сервера Google — `142.250.74.46`. Запомнить нереально. Поэтому придумали **доменные имена**: `google.com` — красиво, понятно, запоминаемо.

Но компьютеры работают только с IP. Кто-то должен переводить `google.com` → `142.250.74.46`.

Этот "кто-то" — **DNS (Domain Name System)**.

---

## DNS — это телефонная книга интернета

Когда ты вводишь `google.com`:

```
1. Браузер → спрашивает локальный DNS-сервер (обычно у провайдера)
2. Если не знает → спрашивает корневой DNS-сервер
3. Тот → направляет к серверу зоны .com
4. Тот → знает где google.com → возвращает IP
5. Браузер получает IP → соединяется с сервером
```

Весь этот процесс занимает **несколько миллисекунд**. Результат кешируется, чтобы не делать это каждый раз.

---

## Структура домена

```
https://  mail  .  google  .  com  /inbox
          ↑         ↑          ↑
       субдомен   домен      зона (TLD)
```

- **TLD (Top Level Domain):** `.com`, `.ru`, `.org`, `.io`
- **Домен:** `google`, `yandex`, `github`
- **Субдомен:** `mail`, `api`, `docs` — разные части одного сайта

---

## Зачем это знать

Когда ты разрабатываешь приложение и настраиваешь его на сервере — тебе нужно:
1. Купить домен
2. Создать DNS-запись (A-запись) указывающую на IP сервера
3. Настроить HTTPS-сертификат

AI поможет с командами, но понимание процесса избавит от растерянности.

---

## 🎯 Главное

> DNS переводит читаемые доменные имена (`google.com`) в IP-адреса. Без этого нам пришлось бы запоминать числа.
