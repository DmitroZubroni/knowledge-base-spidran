## Зачем вообще база данных

Программа в памяти — это как рабочий стол: быстро, удобно, но при выключении компьютера всё исчезает. База данных — это шкаф: медленнее, но данные хранятся постоянно.

Любое серьёзное приложение хранит данные в базе: пользователей, заказы, сообщения, транзакции.

---

## Реляционные базы данных (SQL)

Данные хранятся в **таблицах** — как в Excel. Таблицы связаны друг с другом.

```
Таблица users:          Таблица orders:
id | name | email       id | user_id | amount
1  | Дима | d@test.com  1  | 1       | 500
2  | Аня  | a@test.com  2  | 1       | 1200
```

Для работы используется язык **SQL** (Structured Query Language):
```sql
SELECT name, amount FROM users
JOIN orders ON users.id = orders.user_id
WHERE amount > 1000
```

*Примеры: PostgreSQL, MySQL, Oracle, Microsoft SQL Server*

**Когда использовать:** структурированные данные, связи между сущностями, нужны транзакции (банки, e-commerce).

---

## NoSQL — когда таблиц недостаточно

**Документные БД (MongoDB)** — хранят данные как JSON-документы. Гибкая схема, не нужно заранее определять структуру.

**Ключ-значение (Redis)** — сверхбыстрое хранение в памяти. Используется как кэш.

**Колоночные (Cassandra, ClickHouse)** — оптимизированы для аналитики и больших объёмов данных.

**Графовые (Neo4j)** — для данных с множеством связей (социальные сети, рекомендации).

---

## Как выбрать

| Задача | Выбор |
|--------|-------|
| Пользователи, заказы, транзакции | PostgreSQL |
| Кэширование, сессии | Redis |
| Логи, аналитика | ClickHouse, Elasticsearch |
| Гибкая структура документов | MongoDB |
| Социальные связи | Neo4j |

---

## Итог

- SQL = таблицы, связи, транзакции — классика для большинства задач
- NoSQL = гибкость, масштаб, специализация
- Redis = кэш в памяти, очень быстро
