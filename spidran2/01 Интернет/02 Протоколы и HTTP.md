## Протокол — это договорённость о формате

Протокол = набор правил по которым два участника обмениваются данными. Как правила дорожного движения — все знают что означает красный свет, поэтому движение не хаос.

Стек интернет-протоколов (от нижнего к верхнему):

```
HTTP/HTTPS   ← передача веб-страниц и API
    TCP/UDP  ← надёжная / быстрая доставка пакетов
       IP    ← адресация и маршрутизация
  Ethernet   ← физическая передача бит
```

---

## HTTP — язык браузера и сервера

HTTP (HyperText Transfer Protocol) — протокол по которому браузер запрашивает контент, а сервер его отдаёт.

**Запрос:**
```http
GET /news HTTP/1.1
Host: example.com
Accept: text/html
```

**Ответ:**
```http
HTTP/1.1 200 OK
Content-Type: text/html

<html>...</html>
```

---

## HTTP-методы — глаголы запроса

| Метод | Смысл | Пример |
|-------|-------|--------|
| `GET` | Получить данные | Открыть страницу |
| `POST` | Отправить новые данные | Зарегистрироваться |
| `PUT` | Заменить данные целиком | Обновить профиль |
| `PATCH` | Изменить частично | Сменить аватар |
| `DELETE` | Удалить | Удалить аккаунт |

---

## Коды ответа

| Диапазон | Значение | Примеры |
|----------|---------|---------|
| 2xx | Успех | 200 OK, 201 Created |
| 3xx | Перенаправление | 301 Moved, 304 Not Modified |
| 4xx | Ошибка клиента | 404 Not Found, 401 Unauthorized |
| 5xx | Ошибка сервера | 500 Internal Error, 503 Unavailable |

---

## HTTPS = HTTP + шифрование

Буква **S** — Secure. Данные шифруются с помощью **TLS** до отправки. Без этого провайдер, публичный Wi-Fi или злоумышленник видят весь трафик в открытом виде.

Замочек в адресной строке браузера = HTTPS активен.
