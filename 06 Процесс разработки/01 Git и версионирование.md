## Проблема без версионирования

Ты написал функцию, всё работало. Добавил новую — всё сломалось. Сохранил файл несколько раз — назад не вернуться. В команде из 5 человек — все правят один файл, постоянные конфликты.

**Git** решает обе проблемы.

---

## Что такое Git

**Git** — система контроля версий. Хранит полную историю изменений кода. В любой момент можно:
- Вернуться к любому предыдущему состоянию
- Посмотреть кто и когда изменил строку
- Работать над фичей в изоляции от остальной команды

---

## Ключевые концепции

**Репозиторий (repo)** — проект + вся история изменений

**Коммит** — снимок состояния кода с описанием:
```
commit 3f2a1b4
Author: Дмитрий
Date:   Mon Jun 23

    feat: добавить авторизацию через JWT
```

**Ветка (branch)** — параллельная линия разработки:
```
main   ─────●────────────────────●── стабильный код
             \                  /
feature       ●──●──●──●──●────   разработка фичи
```

**Merge** — влить ветку обратно в main

---

## Командный процесс

1. `git checkout -b feature/login` — создать ветку
2. Написать код, делать коммиты
3. `git push` — отправить ветку на GitHub
4. Открыть **Pull Request** — попросить review
5. Коллеги проверяют код, оставляют комментарии
6. После одобрения — merge в main
7. CI/CD деплоит новую версию

---

## GitHub — облако для кода

**GitHub** (и GitLab, Bitbucket) — хостинг git-репозиториев. Здесь живёт код, происходят code review, запускается CI.

---

## Базовые команды

```bash
git clone <url>          # скачать репозиторий
git pull                 # получить последние изменения
git add .                # подготовить изменения
git commit -m "message"  # сохранить снимок
git push                 # отправить на GitHub
git checkout -b <name>   # создать ветку
git status               # что изменилось
git log --oneline        # история коммитов
```
