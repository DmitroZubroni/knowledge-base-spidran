## Почему именно Java?

Java появилась в 1995 году и до сих пор входит в топ-3 самых используемых языков в мире. Особенно популярна в enterprise — крупном бизнесе, банках, государственных системах.

Причины:
- **Стабильность** — Java держит обратную совместимость: код написанный 20 лет назад работает сегодня
- **JVM** — работает везде, где есть виртуальная машина
- **Строгая типизация** — меньше сюрпризов в production
- **Огромная экосистема** — решение для любой задачи уже существует

## JDK, JRE, JVM — что это всё?

```
JDK (Java Development Kit)
├── JRE (Java Runtime Environment)
│   └── JVM (Java Virtual Machine)  ← запускает байткод
└── Инструменты разработчика (компилятор, отладчик и др.)
```

- **JVM** — запускает байткод, не зависит от ОС
- **JRE** — JVM + стандартные библиотеки (нужно пользователям)
- **JDK** — JRE + инструменты разработки (нужно разработчикам)

## Экосистема вокруг Java

**Spring Framework** — самый популярный фреймворк для создания backend-приложений на Java. Если ты видишь вакансию "Java-разработчик" — там почти всегда есть Spring.

**Maven / Gradle** — инструменты сборки. Автоматически скачивают нужные библиотеки и собирают проект.

**Hibernate / JPA** — работа с базами данных без написания SQL вручную.

**Kafka, RabbitMQ** — очереди сообщений между сервисами.

---

## Kotlin — современная альтернатива

**Kotlin** — язык от JetBrains, работает на той же JVM. Более современный синтаксис, меньше кода, те же возможности. Стал стандартом для разработки Android-приложений и набирает популярность в backend.

---

## Итог

- Java = стабильность, экосистема, enterprise
- JVM = магия "запускается везде"
- Spring = стандарт для backend на Java
- Kotlin = современный Java-совместимый язык
