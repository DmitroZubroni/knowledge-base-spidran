## Проблема: люди не запоминают числа

IP-адрес Google — `142.250.74.46`. Никто не будет это вводить. Поэтому существуют **доменные имена**: `google.com`.

Но компьютеры работают только с IP. Кто-то должен переводить имя в адрес. Этим занимается **DNS (Domain Name System)** — распределённая база данных на тысячах серверов по всему миру.

---

## Как DNS разрешает имя

```
Ты вводишь: google.com
      ↓
1. Браузер проверяет свой кеш → не нашёл
2. Спрашивает DNS-сервер провайдера → не знает
3. Тот спрашивает корневой DNS (.) → знает где .com
4. Запрос идёт к серверу зоны .com → знает где google.com
5. Возвращает IP: 142.250.74.46
6. Браузер кеширует ответ → соединяется с сервером
```

Весь этот процесс — ~20–100 миллисекунд.

## Анатомия доменного имени

```
https://api.example.com/users
         ↑      ↑      ↑
    субдомен  домен   TLD
```

- **TLD** (Top Level Domain): `.com`, `.ru`, `.io`, `.org`
- **Домен**: `google`, `yandex`, `github`
- **Субдомен**: `api`, `mail`, `docs` — разные сервисы одного домена

---

## DNS-записи

При покупке домена ты настраиваешь записи:

| Тип | Для чего |
|-----|---------|
| **A** | Домен → IPv4 адрес |
| **AAAA** | Домен → IPv6 адрес |
| **CNAME** | Псевдоним для другого домена |
| **MX** | Куда направлять email |
| **TXT** | Текст (верификация, SPF для почты) |
