## Проблема: компьютеры говорят на языке цифр

Компьютеры находят друг друга по IP-адресам — например, `142.250.185.46`. Запомнить такое невозможно. Поэтому придумали **доменные имена**: `google.com`, `github.com`.

Но компьютерам всё равно нужен IP. Кто-то должен переводить имена в цифры.

---

## DNS — это и есть переводчик

**DNS** (Domain Name System) — система, которая переводит доменное имя в IP-адрес.

Когда ты вводишь `google.com` в браузере:
1. Браузер спрашивает DNS-сервер: "Какой IP у google.com?"
2. DNS-сервер отвечает: "142.250.185.46"
3. Браузер идёт на этот IP и скачивает страницу

Всё это происходит за миллисекунды, ты этого не замечаешь.

## Структура домена

```
https://blog.company.com/articles/intro

  https://          — протокол
  blog.             — поддомен
  company.          — домен второго уровня (сам сайт)
  com               — домен первого уровня (зона)
  /articles/intro   — путь на сервере
```

Домены как адреса: `com` — страна, `company` — город, `blog` — улица.

---

## Кто управляет DNS

DNS — это не один сервер, это иерархическая система из тысяч серверов по всему миру. Они знают ответы на разные части запроса и передают запрос дальше по цепочке.

Твой провайдер обычно предоставляет тебе свой DNS-сервер. Можно использовать публичные: Google (8.8.8.8) или Cloudflare (1.1.1.1).
