## Две стратегии: перевести заранее или на ходу

Есть два способа превратить читаемый код в инструкции для процессора:

### Компиляция
Весь код переводится в машинный язык **заранее**, до запуска. Получается исполняемый файл (.exe на Windows). Быстро работает, но нужно компилировать отдельно под каждую платформу.

*Примеры: C, C++, Go, Rust*

### Интерпретация
Код читается и выполняется **построчно во время запуска**. Медленнее, но проще запускать где угодно.

*Примеры: Python, Ruby, JavaScript в браузере*

---

## Java: лучшее из двух миров — JVM

Java придумала хитрый способ. Код компилируется не в машинный язык, а в промежуточный — **байткод**. Этот байткод затем запускает **JVM** (Java Virtual Machine) — специальная программа.

```
Твой код (.java)
      ↓ компилятор
   Байткод (.class)
      ↓ JVM
  Любой компьютер
```

**Результат:** "Write once, run anywhere" — написал один раз, запускается на Windows, Linux, Mac, Android — везде, где есть JVM.

JVM ещё и умная: она **JIT-компилирует** (Just-In-Time) горячие участки кода прямо во время работы программы, поэтому Java быстрее чистых интерпретируемых языков.

---

## Что такое рантайм

**Runtime** (рантайм) — это среда выполнения программы. JVM для Java, Node.js для JavaScript на сервере, интерпретатор для Python.

Когда говорят "нужно установить Java" — имеют в виду установить JVM (рантайм), чтобы Java-программы могли работать.
